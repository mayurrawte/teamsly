How detailed is the /install page?

SELF_HOSTING.md has the full guide — Azure AD registration, all 11 Graph API permissions, env vars, deployment options. How much of that goes on the web page?

A

Full guide — everything inline

Convert all of SELF_HOSTING.md to HTML: step-by-step Azure AD registration, the full permissions table, env var reference, and deployment options (Vercel, Fly, Render, bare Node). One long page, no external links needed.

Pros

  • Zero friction — self-hosters never leave the site
  • Looks serious and production-ready

Cons

  • Longest to write
  • Must keep in sync with SELF_HOSTING.md
B

Overview + collapsible steps

Three clear phases (Azure AD setup → env vars → deploy), each with a summary. Details are in expandable <details> blocks — e.g. the permissions table is hidden until you open "Add API permissions". Skimmable for people who know Azure; detailed for people who don't.

Short page: 3 steps, env var block, deploy command. "For the full Azure AD setup guide, see SELF_HOSTING.md on GitHub." Minimal maintenance, lowest effort.

Pros

Cons

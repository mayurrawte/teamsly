Product

Slack mechanics, Microsoft data.

Teamsly does not store conversations. It signs users in with Microsoft Entra ID, proxies Graph API calls through server routes, and renders a Slack-like interface over the data already living in Microsoft Teams.

#

Channels and DMs

Workspace rail, sidebar navigation, unread badges, presence dots, and direct message lists.

T

Threads and reactions

Thread panel, optimistic replies, six Teams reaction types, and rollback toasts on failure.

K

Fast switching

Search modal, Cmd/Ctrl+K jump switcher, proper timestamps, grouping, and date dividers.

G

Graph-first

Client components never call Graph directly. Access tokens stay server-side in API routes.

Architecture

Built for a no-login preview and a real tenant path.

Demo mode Mock data powers /demo for QA without Azure AD.
App shell Shared pure UI components accept real or mock data by props.
Graph proxy Server API routes call Microsoft Graph with session access tokens.

Launch readiness

What is already in place.

  • Priority 1-4 product scope completed in the app.
  • Production dependency audit reports zero vulnerabilities.
  • Filtered TypeScript check is clean except the documented auth config mismatch.
  • Demo route returns HTTP 200 and compiles cleanly.
  • Recoverable failures surface visible toasts instead of silent failure.
`/demo` is the current verification path. It uses mock teams, channels, DMs, reactions, threads, presence, search, and launch UI without requiring Microsoft auth.

Next steps

Owner checklist before public launch.

  1. Resolve the known production build decision. Keep, suppress, or refactor the `tenantId` type mismatch only after confirming real Azure sign-in.
  2. Run real Microsoft Graph smoke tests. Verify sign-in, teams, channels, chats, messages, sends, reactions, replies, and presence.
  3. Do visual QA on the demo. Check desktop, tablet, and mobile widths for overflow, overlap, and interaction polish.

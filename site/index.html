What you get

A complete Teams experience designed for focus — everything mapped onto your existing data, nothing extra.

Channels and direct messages

Workspace switcher, sidebar navigation, unread counts, and presence indicators backed by Microsoft Graph.

Threads and reactions

Slide-in thread panel, optimistic replies and reactions, and clear rollback when a Graph call fails.

Search and Cmd+K

Search across channels, DMs, and loaded messages. A keyboard switcher gets you anywhere in two keystrokes.

Rich messages

Inline mentions, code blocks with copy-to-clipboard, and file attachment cards rendered safely.

Density and preferences

Compact and comfortable density modes, per-user preferences stored locally, and a clean settings modal.

Optional AI features

AI unread summaries, multi-tenant account switcher, and smart browser notifications — behind a feature flag.

How it works

Teamsly never holds your conversations. It signs users in with Microsoft Entra ID and proxies every Microsoft Graph call through server routes, so access tokens never reach the browser.

1

Sign in

NextAuth handles the OAuth flow with your Azure AD tenant. Sessions live in an HTTP-only cookie.

2

Server proxy

API routes call Microsoft Graph using the session's access token. The browser never sees the token.

3

Render

Client components render channels, messages, threads, and presence from the data the proxy returns.

A no-auth demo at /demo swaps the Graph proxy for mock data so you can preview the UI without an Azure tenant.

Self-host

One repository. No managed service to depend on. 

git clone https://github.com/mayurrawte/teamsly
cd teamsly
npm install
cp .env.example .env.local
npm run dev

Fill in .env.local with your Azure AD app credentials: 

AZURE_AD_CLIENT_ID=your-client-id
AZURE_AD_CLIENT_SECRET=your-client-secret
AZURE_AD_TENANT_ID=common
AUTH_SECRET=openssl rand -base64 32
NEXTAUTH_URL=http://localhost:3000

Then open http://localhost:3000 and sign in, or visit /demo to preview the UI without auth. For production, any Node 20+ host works (npm run build && npm start) and Vercel deploys cleanly with the same environment variables.

FAQ

Does Teamsly store my messages?

No. Conversations stay in Microsoft 365. Teamsly only holds an authenticated session token, server-side, for the duration of the request.

Do I need an Azure AD tenant?

For the real app, yes — Teamsly authenticates through Microsoft Entra ID. For a UI preview without auth, run /demo.

Is it really open source?

Yes. AGPL-3.0. The full source is on GitHub. Free to self-host, modify, and contribute. Commercial hosting requires a separate license.

Is there a mobile app?

Not yet. The web app is responsive and works in mobile browsers; a desktop wrapper (Electron) is on the roadmap.

What's the stack?

Next.js 15, NextAuth, Zustand, the Microsoft Graph SDK, Tailwind, and Radix primitives. The optional AI summary route uses the Anthropic SDK.

Self-hosting guide

Run your own Teamsly

Clone the repo, add your Azure AD credentials, and you have a fully self-hosted Microsoft Teams client. Any Node 20+ host works — local, Vercel, Fly, Render, or Docker.

Quick start

Three commands to a running local instance.

1

Clone & install

git clone https://github.com/mayurrawte/teamsly.git
cd teamsly
npm install
2

Configure

cp .env.example .env.local

Fill in .env.local with your Azure AD app credentials:

AZURE_AD_CLIENT_ID=your-application-client-id
AZURE_AD_CLIENT_SECRET=your-client-secret-value
AZURE_AD_TENANT_ID=common
AUTH_SECRET=                 # openssl rand -base64 32
NEXTAUTH_URL=http://localhost:3000
3

Run

npm run dev

Open http://localhost:3000 and sign in with your Microsoft 365 account. Visit /demo to preview the UI without auth.

You need an Azure AD app registration. Teamsly authenticates through Microsoft Entra ID. Before sign-in works you need an app registration with the correct delegated Graph API permissions (User.Read, ChannelMessage.Read.All, Chat.ReadWrite, and others).

Full Azure AD setup guide →

Deploy to production

Set the same five environment variables on any host and run npm run build && npm start.

Vercel

One-click deploy. Import the repo, set the five env vars in the dashboard, done. Auto-deploys on push to main.

Deploy to Vercel

Fly / Render / Railway

Any Node 20+ platform works. Set the five env vars and run the build + start commands. See the full deployment guide for platform-specific notes.

Deployment notes on GitHub →

Bare Node

Build once, serve forever. Node 20+ required. Set NEXTAUTH_URL to your public URL before building.

npm run build
npm start

Troubleshooting

Auth redirect mismatch — sign-in fails immediately

Your NEXTAUTH_URL must match the redirect URI registered in Azure AD. Go to Azure Portal → your app → Authentication → add https://your-domain.com/api/auth/callback/microsoft-entra-id as a Web redirect URI.

Graph API returns 403 after sign-in

Admin consent hasn't been granted for the required delegated permissions. Go to Azure Portal → your app → API permissions → click Grant admin consent for [your tenant]. You need Global Admin rights (or ask your tenant admin).

macOS Gatekeeper blocks the desktop app

The desktop build is unsigned. Run once to clear the quarantine flag and apply an ad-hoc signature:

sudo xattr -cr /Applications/Teamsly.app
sudo codesign --force --deep --sign - /Applications/Teamsly.app

Alternatively, install via Homebrew — the cask preflight script does this automatically:

brew tap mayurrawte/teamsly
brew install --cask teamsly
